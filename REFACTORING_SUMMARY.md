## 🎨 设计原则

### 单一职责原则
每个组件只负责一个功能模块,保持组件的纯净性。

### 开闭原则
通过 props 和 emits 扩展组件功能,避免修改组件内部代码。

### 依赖倒置原则
组件依赖于抽象的接口(props/emits),而非具体实现。

### 接口隔离原则
组件只暴露必要的 props 和 emits,避免过度设计。

## 📝 使用建议

### 1. 组件命名
- 使用 PascalCase 命名组件
- 文件名与组件名保持一致
- 使用语义化的名称

### 2. Props 定义
- 明确定义类型
- 提供合理的默认值
- 使用 TypeScript 类型检查

### 3. 事件命名
- 使用 kebab-case
- 事件名称要清晰表达意图
- 避免使用通用名称

### 4. 样式管理
- 使用 scoped CSS
- 避免样式污染
- 使用 CSS 变量
